# 06. 팀 챕터 작성 템플릿 (TIPS 제안서용)

> **이 문서는 빈 템플릿이다.** `〈...〉` 부분을 실제 정보로 채우면 제안서 '팀' 챕터 초안이 된다. ([05 갭분석](01_TIPS제안서_갭분석및체크리스트.md)의 P0 "팀 자료" 항목에 대응)
>
> **작성 원칙(심사 관점)**: ① **이 과제와의 연결**을 항상 명시 — 단순 경력 나열이 아니라 "그 경력이 §10.1 R&D 난제를 풀 역량임"을 보여준다. ② **증빙 가능**하게(회사·기간·산출물 구체적으로). ③ 과장 금지 — 없는 역량은 **채용/자문 계획**으로 메운다.
>
> ⚠️ **제도 규정**: 팁스는 **대표자를 연구책임자로 지정**해야 한다(IRIS 등록). 따라서 §3.1 대표에 **연구책임자(R&D 총괄) 역할**을 반드시 포함하고, 평가 '창업기업 전문성' 항목에 맞춰 **본 과제 관련 특허·논문·선행 R&D 이력**을 §4에 채운다. (평가 배점·세부는 [01 §3](01_TIPS제안서_갭분석및체크리스트.md))

## 1. 팀 한 줄 요약 (Executive Summary)
> 심사위원이 처음 보는 3~4문장. "왜 이 팀이 이걸 해낼 적임자인가"를 압축.

〈예: 우리 팀은 추천시스템·데이터 엔지니어링 N년 경력의 개발진과 디저트 유통 도메인 전문성을 결합한 N인 팀이다. 대표는 ○○에서 △△를, CTO는 □□에서 ◇◇를 수행했으며, 본 과제의 핵심인 트렌드 예측·랭킹 R&D를 직접 수행할 역량을 보유한다.〉

- **경력**: 〈회사·기간·직무, 핵심 성과 1~3개 — 수치로〉

## 4. R&D 수행 역량 (심사 핵심)
> "이 팀이 §10.1의 연구 난제를 실제로 풀 수 있다"는 근거. 제품 경력과 별개로 **연구·기술 깊이**를 증명.

| 본 과제 R&D 난제([02·03 §10.1]) | 보유 역량/실적(증빙) | 담당자 |
|----------------------------------|----------------------|--------|
| 트렌드 조기 예측(시계열·확산) | 〈예: 전 직장 수요예측 모델 구축, 관련 논문〉 | 〈CTO〉 |
| 하이브리드 검색 + Learning-to-Rank | 〈예: 검색·추천 랭킹 시스템 경험〉 | 〈…〉 |
| 도메인 NER/신조어 탐지 | 〈예: NLP 프로젝트 경험〉 | 〈…〉 |
| 데이터 파이프라인·MLOps | 〈예: 대용량 수집·배치 운영〉 | 〈…〉 |

- **보유 기술 자산**: 〈사내 라이브러리·기존 코드·특허/출원〉 + **자사 실거래 데이터셋**(딸기로픽: 매출·구매 퍼널·팝업별 브랜드 성과·고객 클러스터·키워드 취향·재구매율 등) → 데이터 해자. + **공동연구 분석 방법론·코드**(상세 = [`07`](07_픽미공동연구_분석방법론.md)): ① 디저트 트렌드 **시계열 분석 기법**(안건 1 베이스라인) ② 가게 **자체 등급 계산식 도출**(안건 2 스코어링 v1) ③ 고객/가게 **클러스터링** ④ **키워드 분석**
- **선행 R&D 이력(정식 공동연구)**: **픽미팀 × 딸기로픽 공동연구**(~2026-06, 최종 발표 완료) — 디저트 **트렌드 분석 대시보드**(카테고리별 검색량 추이·MoM·계절성·키워드 랭킹)와 **베이커리 가게 랭킹 대시보드**(평점·리뷰·지역·노출수) + 비즈니스 인사이트(고객 클러스터링·키워드 취향) 구축. → **본 과제 안건 1·2의 작동 PoC이자 선행 연구개발 결과물**. 증빙: [공동연구 산출물(노션)](https://futuristic-eyelash-ee4.notion.site/x-2f3d10ba068b8034bf82cda2937f0535) · 대시보드 · 최종 발표자료 · 데이터. 〈계약/협약 문서·기간·역할 분담 명시〉

## 5. 조직도 & 인력 운영
- **현재 조직**: 〈대표 아래 R&D팀/데이터팀/사업팀 구성 — 간단 트리 또는 표〉
- **본 과제 전담 인력**: 〈전담 N명 + 겸직 N명〉
- **채용 계획(역량 공백 보완)**: 〈예: 1년차 ML 엔지니어 1명, 데이터 라벨러 N명 — 시점·예산 연계〉

```
〈조직도 예시〉
대표(CEO)
├─ R&D팀: CTO + ML 엔지니어 〈채용예정〉
├─ 데이터팀: 데이터 엔지니어
└─ 사업팀: 입점사 영업/운영
```
