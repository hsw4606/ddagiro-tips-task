# 07. 픽미 공동연구 분석 방법론 (선행 R&D·작동 PoC 증빙)

> **이 문서의 위치**: 안건 1·2의 **선행 연구개발 이력이자 작동 PoC**인 **픽미팀 × 딸기로픽 정식 공동연구**의 *실제 분석 방법론·코드·산출물*을 정리한다. 추상적 주장("공동연구를 했다")이 아니라 **구체적 방법론 4종**으로 입증해, TIPS 심사의 **선행 연구개발 이력(개발 적정성 15점)·검증력(개발 실현성 10점)·데이터 해자(사업화 20점)** 를 뒷받침한다.
>
> ⚠️ **출처·정확도 주의**: 아래 방법·구조는 픽미 공동연구 노션 산출물에서 확인한 것이다. 가중치 상수·코드 라인·정확한 군집 수치 등 **세부 값은 원본(노션)을 정본(正本)으로 삼는다**. 본 문서는 *방법론의 구조와 TIPS 연계*를 정리한 요약이며, 발표·제출 시 수치는 원본을 재확인해 인용한다.

## 1. 디저트 트렌드 시계열 분석 (안건 1 베이스라인)

**증빙**: [신규 점포 발굴 대시보드 — 디저트 트렌드 시계열 분석 방법](https://futuristic-eyelash-ee4.notion.site/_-2f4d10ba068b80dfb57edb845cd7c9dd)

- **목적**: 지역·카테고리별 디저트 검색량/관심도의 **시간적 추이**를 분석해 성장·쇠퇴 신호와 계절성을 포착한다(신규 점포 발굴/입지 판단의 근거).
- **방법(요지)**: 시계열 데이터 수집 → 카테고리별·지역별 추이 라인/스파이크 차트화 → **상승·급등·정점·계절 주기 패턴** 식별. (다수의 카테고리에 대해 동일 파이프라인을 반복 적용한 다중 시계열 차트 세트가 산출물)
- **산출물**: 카테고리별 검색량 추이 시계열 차트군 + 패턴 해석.
- **본 과제 고도화(안건 1)**: 단순 추이 시각화 → **"정점 전 성장기 진입"을 리드타임 ≥ 14일로 조기 탐지**하는 알고리즘(S-curve 피팅 + 1·2차 미분 모멘텀 + 과거 곡선 시계열 유사도)으로 발전. 이 공동연구 시계열 코드는 **고도화 출발점(선행 자체기법 v1)** 이며, 백테스팅 *개선폭*은 별도의 **단순 비교군(이동평균 임계, [guides/09 §6.1](guides/09_트렌드스코어링_백테스팅.md))** 대비로 측정한다(선행 v1은 출발점이지 백테스트 비교군이 아님). → [02 §10.1](02_트렌드분석추천시스템_기획.md)

### 1.1 트렌드 점수 산출 공식 (대시보드 PoC v1 — "점수 기준"의 실체)

공동연구 PoC 대시보드는 단순 언급량이 아니라 **여러 신호를 가중 합산한 "트렌드 점수"** 로 디저트를 줄세운다. 이것이 *"점수를 무슨 기준으로 매기나"* 에 대한 딸기로픽의 자체 노하우이자, 안건 1 KPI([02 §10.1](02_트렌드분석추천시스템_기획.md))의 베이스라인 출발점이다.

| 신호 | 가중치(PoC) | 의미 |
|------|------------|------|
| 검색량 증가율 | 30% | 실수요(네이버 데이터랩·구글 트렌드) |
| SNS 언급 증가 | 25% | 바이럴 확산(인스타·유튜브) |
| 감성(긍정도) | 20% | 호감도(단순 빈도 보정) |
| 재구매 의향 | 15% | 반짝/지속 구분 |
| 인구통계 적합 | 10% | 타깃 연령·성별 부합 |

> ⚠️ 가중치는 대시보드 페이지/목적별로 미세조정된다(예: 키워드 순위 페이지 = 검색 30·SNS 20·긍정 20·재구매 15·인구 10·기타 5). 초기 GPT 단계에선 실시간성 중심(최근 24h SNS+콘텐츠 ≈ 60%)으로 가중을 달리한 변형(Today Trend Score)도 시험. **정확 상수는 PoC/노션 원본을 정본으로 인용.**
> **시간창 다중 비교(노하우)**: Today(24h·최우선) / Short(7일) / Baseline(직전 7일·비교) / Long(3개월·지속성) 을 함께 보아 *조기성*과 *지속성*을 분리 판정.
> **다신호 교차검증(노하우)**: 예) "SNS 바이럴↑인데 재구매 의향↓ → 기대 불일치, 주의" 식 경보로 **반짝 유행(오탐)** 을 거른다(안건 1 §10.1 난제 3).

본 과제는 이 **고정 가중합(v1)** 을 S-curve·모멘텀·시계열 유사도 기반 *정점 전 조기탐지* + 운영 피드백 가중치 학습으로 고도화한다(개선폭 = KPI). 쉬운 설명·근거 연결: [02 §5.3.1](02_트렌드분석추천시스템_기획.md).

## 3. 가게 자체 등급 계산식 도출 (안건 2 스코어링 v1)

**증빙**: [신규 점포 발굴 대시보드 — 가게 자체 등급 계산식 도출 방법](https://futuristic-eyelash-ee4.notion.site/_-2f4d10ba068b808d8403e0447e7f4440)

- **목적**: 외부에 의존하지 않는 **자체 가게 종합 등급**을 산출해 "발굴할 만한 가게"를 객관적으로 줄세운다.
- **방법(요지)**:
  1. **입력 지표**: 평점(rating)·리뷰 수·카카오맵 평점 등 공개 가게 지표.
  2. **점수 산출**: 지표를 정규화/가중합해 **종합 점수(score)** 도출. 분포는 `describe()` 통계표(count·mean·std·min·25%·50%·75%·max)로 점검.
  3. **등급화**: 순위 구간(예: 1～10 / 11～30 / 31～50 / 51～100위)별로 구간 점수를 부여하고, **분위(quantile) 기반 컷오프**로 최종 **S～D 5등급** 부여.
  > 구간 경계·가중치·구간별 점수 등 **정확한 상수는 노션 원본을 정본으로 인용**한다.
- **산출물**: 가게별 종합 점수·등급 테이블 + 등급 산식 정의.
- **본 과제 고도화(안건 2)**: 이 등급 산식을 **랭킹 고도화 출발점(선행 자체기법 v1)** 으로 삼고, 본 과제에서 **질의 적합도 1차 + 품질·평판/지역·배송/트렌드(안건 1 연계) 가산**의 하이브리드 랭킹·Learning-to-Rank로 고도화한다(랭킹 품질 *개선폭*은 골든셋에서 단순 비교군 "키워드+단일 임베딩" 대비로 측정 — [03 §10.1](03_자연어추천랭킹시스템_기획.md)). → [guides/09](guides/09_트렌드스코어링_백테스팅.md)

## 4. 클러스터링 분석 (안건 2 세그먼트·개인화)

**증빙**: [비즈니스 대시보드 — 클러스터링 분석 방법](https://futuristic-eyelash-ee4.notion.site/_-2f4d10ba068b8089b113ddaf29b5ccc6)

- **목적**: 가게/고객을 특성으로 **군집화**해 비슷한 집단별 특징·전략을 도출(타깃팅·개인화·입점 제안).
- **방법(요지)**: 피처 정규화(스케일링) → 비지도 군집화로 **4개 군집(Cluster 0～3)** 도출 → 군집별 **프로파일(핵심 특징·주요 키워드·운영/마케팅 전략)** 작성. 노션 원본에 군집화 Python 코드와 군집별 통계·키워드 표 수록.
- **산출물**: 군집 4종 프로파일 카드(특징·키워드·전략).
- **본 과제 고도화(안건 2)**: 군집을 **추천 개인화·세그먼트별 랭킹 가중**과 **입점 타깃팅(어떤 가게에 무엇을 제안)** 에 활용. 딸기로픽 자사 고객 클러스터·키워드 취향 데이터와 결합해 정밀화. → [03 §10.3](03_자연어추천랭킹시스템_기획.md)

## 5. TIPS 심사 활용 포인트 (요약)

- **개발 적정성 — 선행 연구개발 이력(기술성 15)**: 위 4종은 *수행 완료된* 자체 분석 방법론·코드. "선행 R&D 없음"이 아니라 **있고, 본 과제는 그 고도화**임을 입증. → [01 §4](01_TIPS제안서_갭분석및체크리스트.md)
- **개발 실현성 — 검증(기술성 10)**: 시계열 코드(§1)·등급 산식(§3)은 곧바로 **백테스팅/평가 베이스라인**이 된다. *"개선폭"* 으로 R&D 가치를 정량화. → [02·03 §10.1]
- **사업화 — 데이터 해자(사업성 20)**: 딸기로픽 자사 실거래 데이터 + 공동연구 분석 자산(4종) = **복제 난도 높은 결합 자산**. → [04 §2](04_인터넷데이터_활용방법.md)
- **"단순 LLM API 통합" 반박(기술성 전반)**: 핵심 알고리즘(시계열 예측·등급·군집·키워드)이 **이미 코드로 존재** → LLM은 부품, R&D는 우리 알고리즘 고도화. → [01 §6.1](01_TIPS제안서_갭분석및체크리스트.md)
- **권리화(지식재산)**: 이 방법론들은 특허로도 출원됐다(인기도 지수·AI 수요예측·통합추천·플랫폼 4건) → 방법론·코드(본 문서) + 권리(특허)의 이중 증빙. → [08 특허출원·지식재산](08_특허출원_지식재산.md)

> **남은 일**: ① 공동연구 **계약/협약 문서·기간·역할분담** 명시(증빙 보강) ② 시계열·등급 코드(선행 자체기법 v1 = 고도화 출발점)를 **재현 가능하게 정리**하고, 단순 비교군 대비 개선폭을 측정(제3자 공인 대비) → [01 §7](01_TIPS제안서_갭분석및체크리스트.md).
