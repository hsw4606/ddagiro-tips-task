## 2. 평가 방식·일정·발표 주체 (⚠️ 중요 정정)

- **선정평가 = 서면평가 + 대면평가(현장 발표 원칙, 사전 희망 시 온라인 허용)**.
- **발표 50분 내외**: 준비 5분 / **발표 20분 / 질의응답 20분**.
- **발표 주체는 "운영사 대표자 또는 투자담당 심사역"이 원칙.** 추천 창업기업의 **연구책임자 등은 보조 발표** 가능(기술 설명). → 피치덱·발표는 **운영사와 공동 준비**가 전제다.
- **사전질의 제도**: 평가 30분 전 사전질의서 발송 → 발표 질의응답에서 답변.
- **연구책임자 = 대표자**: 팁스는 **대표자를 연구책임자로 지정**해야 함(IRIS 등록 시). 주관기관 기관유형 **<중소기업>** 등록 필요.
- **접수·관리 시스템**: 과제접수 **IRIS**(iris.go.kr), 연구비관리 **RCMS**(rcms.go.kr). 가점은 **IRIS에서 신청(체크)+증빙 업로드 필수**(개별보완 불인정).
- *해당 회차(26-2차) 평가일정 예시: '26.6.9～6.17 중 1일. **차기 회차 일정·마감은 공고/운영사로 확인.***

## 5. 가점 전략 (최대 5점 — IRIS 신청+증빙 필수)

> 종합평점에 **최대 5점** 가산. 점수 경합 시 결정적. **예비창업자는 가점 불가 → 법인 설립 후 추천.** 각 항목 **증빙을 IRIS에 업로드**(개별보완 불인정).

| 가점 항목 | 점수 | 조건/증빙 | 딸기로 적용 검토 |
|-----------|------|-----------|------------------|
| 지역균형(본사 비수도권) | 3 | 본사 서울/경기/인천 외 + 잔류확약서, 등기부 기준 | ❌ **해당 없음** — 본사 서울 동작구(수도권) |
| 지역균형(공장 비수도권) | 2 | 공장 비수도권 + 공장등록증 | ❌ **해당 없음** — 자체 제조시설 없음(중개·배송 플랫폼, 공장등록증 부재) |
| 일자리 창출 | 1 | 내일채움공제 가입(접수마감일 기준 운영, 1년 유지) | 🟡 고용계획과 연계 검토 |
| 연계사업 우수성과 | 2 | 프리팁스 성과평가 '성공' 공문 | ❓ 프리팁스 이력 시 |
| 사회적 가치(ESG) | 2 | 기후테크·소셜벤처 등 — **BM과 직접 연계**(단순 선언 불인정), 운영사 ESG 사유서 | 🟡 친환경 포장·소상공인 상생 등 BM 연계 가능성 검토 |
| 벤처/이노비즈 인증 | 최대1 | 벤처기업·이노비즈 확인서 | 🟡 인증 보유/취득 검토 |
| 퇴직연금(기업형 DC/DB/IRP) | 1 | 도입 통지서·가입확인서(개인형 IRP 불인정) | 🟡 도입 검토 |

> **⚠️ 제약**: 본사 서울(동작구)·자체 제조시설 없음 → **지역균형 5점(본사3+공장2) 전부 불가.** 가점 5점은 아래 **취득형 항목 스태킹**으로만 가능하다.

**확보 가능 풀(지역균형 제외)** = 일자리 1 + 연계사업(프리팁스) 2 + ESG 2 + 벤처/이노비즈 1 + 퇴직연금 1 = **최대 7점 → 5점 캡**.

- **시나리오 A (프리팁스 이력 있음)**: 프리팁스 성과 '성공' 2 + ESG 2 + 일자리 1 = **5점** (가장 쉬움).
- **시나리오 B (프리팁스 없음)**: ESG 2 + 일자리(내일채움공제) 1 + 벤처/이노비즈 1 + 퇴직연금 1 = **5점** (4개 항목 모두 취득·증빙 필요 → 난도 높음, 조기 착수 필수).

→ **액션**: ① 프리팁스 이력 여부 즉시 확인(있으면 2점 확보로 5점 크게 수월) ② **ESG 2점은 BM 직접 연계 필수**(친환경 포장·소상공인 상생 — 단순 선언 불인정) → 운영사 ESG 사유서 준비 ③ 내일채움공제·퇴직연금·벤처/이노비즈는 **접수마감일 기준 충족**해야 하므로 일정 역산해 조기 가입·신청.

## 6. 핵심 리스크 & 대응

**6.1 "그냥 LLM API 쓰는 거 아닌가?" (기술성 직결)**
정부 R&D는 연구개발 과제여야 한다. `02·03 §10.1`에 **난제 정의 + 정량 KPI + 자체 기법 + 검증법**을 명시해 *"LLM은 부품, 핵심은 우리 예측·랭킹·검증 알고리즘"* 으로 포지셔닝(이미 반영). **이 주장은 말이 아니라 선행 코드로 입증된다** — 픽미 공동연구의 **시계열 분석 기법·가게 자체 등급 계산식**(자체 알고리즘 v1, 방법론 4종 상세 = [`07`](07_픽미공동연구_분석방법론.md))이 이미 존재하고, 본 과제는 이를 AI 예측·랭킹으로 고도화하는 것. 발표에서 베이스라인(자체 v1) 대비 개선 수치로 설명. **나아가 핵심 기술(인기도지수·AI 수요예측·통합추천)이 특허 4건으로 출원**([`08`](08_특허출원_지식재산.md))되어 있어, 독자 알고리즘임이 권리로도 뒷받침된다.

**6.2 유사과제 중복 → '지원대상 제외' (치명적)**
IRIS는 신청 전 **유사과제 검출**을 한다. 연구과제명·연구책임자·목표·내용·기대효과·키워드를 IRIS 입력값과 동일하게 작성해 **사전 차별성 검토** 필수. 유사 판정 시 **점수와 무관하게 지원 제외**. → 신청 전 운영사와 유사과제 사전조회.

**6.3 글로벌·일자리 항목 (사업성/기대효과)**
글로벌 성장가능성(5)은 **전략 수립 완료**(`02·03 §10.3.1`: 엔진=수출자산·**Pre-Phase 국내 인바운드 실증(방한 외국인·딸기로픽 For Visitors)**·3단계·진입순서 일본→동남아→미국) → 글로벌 KPI(Pre-Phase～3년 차) 기입 완료, 트랙션 확보 후 상향 캘리브레이션만 잔여. 기대효과(일자리, 15)는 아직 약함 → §4 갭대로 보강.

**6.4 발표 주체 = 운영사 (준비 책임 분담)**
발표는 운영사 대표/심사역이 원칙. 연구책임자(=대표)는 기술 보조. → **운영사와 피치덱·사전질의 공동 준비.**

## 7. 트랙션·검증 실행계획 (🔴 계획만 → 🟢 증거 확보)

> **목적**: 트랙션을 메우는 **실행 로드맵**. 심사 '개발 실현성(10)'의 *검증방법 타당성·객관적 입증*(§3)에 직결되며, 동시에 `02·03 §10.1` KPI의 **베이스라인 실측**을 겸한다. 세 수단은 입증 대상이 다르다 — **백테스팅=기술 입증 / MVP=작동 입증 / LOI=수요 입증**.
>
> **✅ 업데이트(2026-06-18)**: 픽미팀과의 **정식 공동연구 산출물**(트렌드 분석·가게 랭킹 대시보드 + 딸기로픽 실거래 데이터)로 **②작동 PoC와 수요/전환 실데이터를 이미 확보**했다. 따라서 잔여 핵심은 **①백테스팅 KPI 실측 1건**(베이스라인 대비 개선율 산출)으로 좁혀진다. ③LOI는 사업화 보강용으로 병행.

| 수단 | 무엇을 입증 | 방법·데이터셋 | 지표(증빙) | 비용/기간 | 우선 |
|------|-------------|----------------|------------|-----------|------|
| **① 콜드스타트 백테스팅** (안건 1) | 예측 알고리즘이 실제로 맞힌다 | 흥망이 끝난 과거 아이템(약과·두바이초콜릿·소금빵·크룽지)의 네이버 데이터랩 과거 1～2년 추이만으로 "성장기 진입"을 사후 예측 → 실제 정점과 대조. **보유데이터 0으로 가능** → [`02 §10.1`](02_트렌드분석추천시스템_기획.md), [`guides/09 §5`](guides/09_트렌드스코어링_백테스팅.md), [`04 §2`](04_인터넷데이터_활용방법.md) | 성장기 탐지 정밀도·재현율, **리드타임(일)**, 오탐률 — 베이스라인(단순 언급량 임계) 대비 개선율. 재현 가능 노트북 + 결과 리포트 | 인터넷 과거데이터 + Claude 추론 / **1～2주** | ⭐ 1순위 |
| **② MVP/PoC 데모** (안건 2) | 시스템이 실제로 작동한다 | 자연어 질의 → 전국 업체 통합 랭킹의 *얇은* 동작 데모. 전국 커버리지 불필요 — **1～2개 도시·카테고리로 한정**한 좁고 깊은 버전 → [`03 §9`](03_자연어추천랭킹시스템_기획.md) | 추천 적합도(사람평가 nDCG/만족도), 응답시간, 입점+비입점 통합 커버리지 수. 발표 20분 라이브 데모 | 개발인력 + Claude Code Max / **2～4주** | 🟢 **일부 기보유**(공동연구 대시보드 = 안건 1·2 작동 PoC) |
| **③ LOI(의향서)** | 시장 수요가 실재한다 | 기존·신규 입점사 N곳에서 "이 기능이 나오면 쓰겠다" 서명 의향서 확보. **운영사와 공동** 진행 시 신뢰도↑ | 서명 LOI(업체명·용도·예상 사용규모) → 사업화전략(20점)·기대효과 보강 | 영업 리드타임 / 상시 | ⭐ 병행 |

**실행 순서**: (a) 백테스팅 1건으로 §10.1 KPI 베이스라인 실측 — **베이스라인 = 픽미 공동연구의 시계열 분석 코드**(자체 v1)로 두고 본 과제 신기법과 비교 → (b) 그 수치를 발표 근거로 사용(§6.1 "왜 R&D인가" 반박) → (c) LOI 2～3건으로 수요 입증 → (d) 여력 시 MVP 데모. **제3자 공인**(§3) 여지: 백테스트 데이터셋·코드를 제출/공개용으로 정리하면 객관성 가점에 유리.
