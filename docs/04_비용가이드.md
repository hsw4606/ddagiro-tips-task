## 3. LLM 비용 워크시트 (가정 → 계산)

> 아래 일 처리 건수 `D`는 막연한 추정이 아니라 §3.0의 **"감시 키워드 수 × 키워드당 신규 건수"** 로 도출한 값이다. 실제 운영 규모가 정해지면 §3.0의 파라미터만 바꿔 재계산한다.

### 3.0 일 처리 건수(D) 산정 근거 — 바닥부터 추정

시스템은 인터넷 전체를 긁지 않고, **감시 대상 키워드/아이템 목록(watchlist)** 에 대해서만 정기 조회한다. 따라서:

> **일 처리 건수 D ≈ (감시 키워드 수 K) × (키워드당 일 신규 유효 건수 n)**

- `K` = 추적할 디저트 아이템·재료·맛·테마 키워드 수. (초기엔 사람이 시드 목록 + 신조어 자동 발견으로 확장)
- `n` = 키워드 1개당 하루에 새로 잡히는 **중복 제거·디저트 도메인 필터 통과** 건수. (네이버 검색 API는 쿼리당 최대 ~100건 반환하나, 신규·유효분만 카운트)

| 시나리오 | K(키워드) | n(키워드당/일) | **D = K×n** | 비고 |
|----------|-----------|----------------|-------------|------|
| **MVP** | ~200 | ~25 | **≈ 5,000/일** | 무료 공식 API 3종 범위 |
| 성장기 | ~500 | ~40 | **≈ 20,000/일** | 유튜브·배달앱·해외 추가 |
| 확장기 | ~1,000 | ~50 | **≈ 50,000/일** | 멀티플랫폼·이미지 트렌드 |

> 즉 MVP의 D=5,000은 "디저트 키워드 약 200개를 매일 모니터링"한다는 **현실적이고 작은** 출발점이다. 한국의 디저트·카페 관련 블로그/뉴스/영상 발생량은 이보다 훨씬 크므로, K·n을 키우면 상한은 충분히 높다(비용은 §3.1·§5에서 선형 증가). **딸기로가 실제로 추적하고 싶은 키워드 규모(K)만 정해주면 D와 전체 비용이 확정된다.**

### 3.1 대량 작업 — 디저트 글 분류 + 감성분석 (Haiku, 배치)
- 가정: 하루 수집·처리 글 수 `D`건, 글당 입력 `~550`토큰, 출력 `~60`토큰.
- 일 입력 토큰 = D × 550, 일 출력 토큰 = D × 60.

| 시나리오 | D(일 처리 건수) | 정가 월비용 | 배치 50%↓ 월비용 |
|----------|----------------|-------------|------------------|
| MVP | 5,000 | ≈ $112 | **≈ $56** |
| 성장기 | 20,000 | ≈ $450 | **≈ $225** |
| 확장기 | 50,000 | ≈ $1,120 | **≈ $560** |

*(계산식 예 — MVP: 입력 5,000×550×30=82.5M×$1 + 출력 5,000×60×30=9M×$5 = $82.5+$45 ≈ $127/월. 캐싱·반올림 반영 시 ~$112. 배치 적용 시 절반.)*

### 3.2 소량 고부가 — 트렌드 인사이트 브리핑 (Sonnet 또는 Opus)
- 가정: 매일 상위 `T`개 아이템 브리핑, 건당 입력 `~3,000`토큰(지표+근거), 출력 `~800`토큰.

| 시나리오 | T(일 브리핑 수) | Sonnet 월비용 | Opus 월비용 |
|----------|----------------|---------------|-------------|
| MVP(주간 위주) | 20 | ≈ $13 | ≈ $22 |
| 성장기 | 50 | ≈ $32 | ≈ $54 |
| 확장기 | 100 | ≈ $63 | ≈ $108 |

### 3.3 LLM 합계 (월, 대략)
| 시나리오 | LLM 월비용(배치+티어링 적용) |
|----------|------------------------------|
| **MVP** | **약 $70 (≈ 10만 원) 내외** |
| 성장기 | 약 $260 (≈ 35만 원) 내외 |
| 확장기 | 약 $670 (≈ 90만 원) 내외 |

> 핵심 메시지: **AI 비용은 생각보다 작다.** 티어링·배치·캐싱을 쓰면 MVP는 월 십수만 원, 본격 확장기에도 월 100만 원 안쪽에서 운영 가능. 비용의 비선형 증가는 "처리 건수 × 고급 모델 남용"에서 오므로 설계로 통제한다.

### 3.4 프로바이더별 MVP 대량처리 월비용 비교
동일 워크로드(MVP, D=5,000건/일, 건당 입력 550·출력 60토큰, 월 30일 = 입력 82.5M·출력 9M)를 **각 사 저가 모델**로 처리할 때:

| 프로바이더·모델 | 정가 월비용 | 배치 50%↓ |
|-----------------|-------------|-----------|
| OpenAI GPT-4.1 nano ($0.10/$0.40) | ≈ $12 | **≈ $6** |
| OpenAI GPT-5.4 Nano ($0.20/$1.25) | ≈ $28 | ≈ $14 |
| Gemini 3.1 Flash-Lite ($0.25/$1.50) | ≈ $34 | ≈ $17 |
| Gemini 3 Flash ($0.50/$3) | ≈ $68 | ≈ $34 |
| OpenAI GPT-5.4 Mini ($0.75/$4.50) | ≈ $102 | ≈ $51 |
| **Claude Haiku 4.5 ($1/$5)** | ≈ $128 | ≈ $64 |

> 시사점: **MVP 대량처리 AI비용은 프로바이더에 따라 월 $6~$64 범위**로, 어느 쪽을 골라도 **사실상 무시할 수준**이다. 따라서 선택 기준은 단가가 아니라 **① 한국어·디저트 도메인 분류 품질, ② 신뢰성/속도, ③ 데이터 처리 약관·리전**이어야 한다. 인사이트(고급) 작업은 건수가 적어 어느 사든 월 수만 원 이하라 차이가 미미하다.

| 시나리오 | 데이터 | LLM | 인프라 | **합계(월)** |
|----------|--------|-----|--------|--------------|

→ **순수 인프라+AI 운영비는 MVP 월 수십만 원, 확장기에도 월 수백만 원 수준**으로, 인건비 대비 작다.

### 5.2 TIPS 지원 규모 (2026년 개편 기준) + 예산 배분

2026년 TIPS R&D는 "창업 초기 소규모 지원"에서 **성장·글로벌까지 아우르는 전주기 체계**로 개편됐다. 트랙별 정부지원 한도(과제당 최대):

| 트랙 | 정부지원 한도(최대) | 대상/성격 |
|------|---------------------|-----------|
| **일반(기본) TIPS R&D** | **8억원** | 창업 초기 — **딸기로가 처음 노릴 트랙** |
| 딥테크 챌린지 | 50억원 | 기본 트랙 성과 기업의 후속 대형 R&D |
| 스케일업 TIPS | 30억원 | 성장 단계 (물량 약 300개) |
| 글로벌 TIPS | 60억원 | 해외 진출 |
| 생태계혁신형 | 200억원 | 대형 협력형 |

- **매칭/민간투자 요건**: TIPS는 **운영사(액셀러레이터)의 선(先)투자 추천**이 전제다. 운영사 엔젤투자를 받은 뒤 정부 R&D가 매칭된다. 딥테크 후속트랙은 정부지원 **최대 75%** + 일반트랙 이후 5억원 이상 후속투자 확보 요건이 명시돼 있다. (일반트랙의 정확한 매칭비율·자부담률은 **해당 연도 공고로 최종 확인** 권장)
- **딸기로 시나리오**: 일반(기본) TIPS R&D **최대 8억원**을 1차 목표로 잡고, 성과 시 스케일업(30억)·딥테크(50억) 후속 트랙으로 확장하는 경로가 자연스럽다.

**예산 배분 — 진짜 큰 항목은 R&D 인건비**
"개발 인력 충분" 전제라도, 8억 규모 R&D 예산의 대부분은 **연구개발 인건비**로 계상되는 게 일반적이다:
- 트렌드 예측 모델/파이프라인 **연구개발 인건비** (최대 비중)
- 데이터 엔지니어링·웹 플랫폼 개발 인건비
- 클라우드 인프라·AI API 사용료 (§1~4: MVP 월 수십만 원 → 2~3년 누적도 수천만 원 내외, 전체 예산 대비 소액)
- (선택) 상용 데이터 솔루션 구독, 외부 자문·법무 검토

> 즉 **운영 비용(AI+인프라)은 예산의 작은 부분**이고, TIPS 8억은 사실상 **"몇 명을 몇 년 투입하느냐"(인건비)**가 좌우한다. 제안서 예산표의 정확한 비목·자부담률·인건비 단가는 **해당 연도 공고 양식 기준으로 확정**할 것.

## 6. 예산을 "결과에 연동"하는 방법 (제안)
1. **MVP 예산은 거의 고정·소액**(월 수십만 원 운영비)으로 잡고 빠르게 검증.
2. **확장 예산은 KPI 트리거에 연동**: 예) "추천 채택률 X% 또는 베타 입점사 Y곳 돌파 시 성장기 인프라로 증액". → 성과 없이 비용만 늘지 않게.
3. **AI 비용은 상한(budget cap) 설정**: 모델 티어·배치·캐싱 + 월 토큰 상한 알림으로 폭주 방지.
4. 실제 수집 건수·트래픽이 정해지면 §3 워크시트 가정값을 교체해 정밀 재산정.

- 월 LLM 입력비 = (일 건수 × 건당 입력토큰 × 30 ÷ 1,000,000) × 모델 입력단가
