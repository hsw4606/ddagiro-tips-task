# 05. 웹 크롤링 기초 & 준법 (제로베이스)

> 목표: 공식 API가 없는 소스(배달앱 인기메뉴 공개 페이지, 해외 매체 등)를 **합법적·안정적으로** 수집한다.
> 선행: [01_개발환경_세팅.md](01_개발환경_세팅.md)
> ⚠️ 크롤링은 법적 리스크가 가장 큰 영역이다. **§1 준법 먼저 읽고 시작**할 것.

## 1. 준법 체크리스트 (시작 전 필수)

- [ ] 공식 API가 있으면 **API 우선** — 크롤링은 최후 수단.
- [ ] 대상 사이트의 **`robots.txt`** 확인 (예: `https://example.com/robots.txt`).
- [ ] 대상 사이트 **이용약관**에 크롤링/자동수집 금지 조항이 있는지 확인.
- [ ] **개인정보**(작성자 식별정보 등) 수집·저장 금지.
- [ ] **원문 전재·재배포 금지** — 집계 지표/메타만 보관(`docs/04` §0,§5).
- [ ] **rate limit**: 요청 간 지연(예 1~3초), 동시요청 자제 — 상대 서버에 부하 금지.
- [ ] 식별 가능한 **User-Agent** 사용(연락처 포함 권장).

> 판단이 애매하면 수집하지 말고, 공식 API/상용 데이터로 대체. TIPS 제안서에도 "준법 수집"을 명시하면 신뢰도↑.

## 2. robots.txt 자동 확인

```python
import urllib.robotparser as rp

def allowed(url: str, user_agent: str = "ddalgiro-bot") -> bool:
    from urllib.parse import urlparse
    base = "{0.scheme}://{0.netloc}".format(urlparse(url))
    parser = rp.RobotFileParser()
    parser.set_url(base + "/robots.txt")
    parser.read()
    return parser.can_fetch(user_agent, url)

if __name__ == "__main__":
    print(allowed("https://example.com/some/page"))
```

## 3. 정적 페이지 — requests + BeautifulSoup

대부분의 단순 HTML 페이지는 이걸로 충분.

```bash
pip install beautifulsoup4 lxml
```

```python
import time, requests
from bs4 import BeautifulSoup

HEADERS = {"User-Agent": "ddalgiro-bot/1.0 (contact: hsw4606@gmail.com)"}

def fetch(url: str) -> str:
    r = requests.get(url, headers=HEADERS, timeout=10)
    r.raise_for_status()
    time.sleep(2)                      # 예의상 지연 (rate limit)
    return r.text

def parse_titles(html: str) -> list[str]:
    soup = BeautifulSoup(html, "lxml")
    # 예시: 페이지 구조에 맞게 selector 지정
    return [el.get_text(strip=True) for el in soup.select("h3.title")]
```

## 4. JS로 그려지는 페이지 — Playwright

배달앱/SPA처럼 자바스크립트로 렌더링되는 페이지는 requests로 안 보인다 → 헤드리스 브라우저.

```bash
pip install playwright
playwright install chromium      # 브라우저 엔진 1회 설치
```

```python
from playwright.sync_api import sync_playwright

def render(url: str) -> str:
    with sync_playwright() as p:
        browser = p.chromium.launch(headless=True)
        page = browser.new_page(user_agent="ddalgiro-bot/1.0")
        page.goto(url, wait_until="networkidle", timeout=20000)
        html = page.content()
        browser.close()
        return html
```

> Playwright는 무겁다. 가능하면 §3(requests)로 해결하고, 꼭 필요한 페이지만 §4 사용.

## 5. 안정적 수집 패턴

- **재시도 + 백오프**: 일시 오류에 지수 간격 재시도.
- **캐시**: 같은 URL 반복 요청 금지(이미 받은 건 저장분 사용).
- **에러 격리**: 한 페이지 실패가 전체 잡을 죽이지 않게 try/except + 로깅.
- **수집량 상한**: 키워드·페이지 수 cap을 두고, 초과분은 로그로 남김(silent truncation 금지).

## 7. 트러블슈팅

- **403/429**: User-Agent·지연·빈도 조정. 그래도 막히면 그 소스는 포기(차단 우회 시도 금지).
- **빈 결과**: JS 렌더링 페이지 → §4 Playwright로 전환.
- **구조 변경**: 사이트 개편 시 selector 깨짐 → 파서를 모듈화해 유지보수.
