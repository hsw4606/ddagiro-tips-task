# 01. 개발 환경 세팅 (제로베이스)

> 목표: 빈 PC에서 시작해 **Python 프로젝트 + 가상환경 + 비밀키 관리 + git**까지 갖춘다. 이후 모든 가이드의 토대.

## 1. Python 3.11+ 설치
macOS(Homebrew 기준):
```bash
# Homebrew 없으면 먼저 설치: https://brew.sh
brew install python@3.12
python3 --version   # Python 3.12.x 확인
```
> Windows는 https://www.python.org 에서 설치(설치 시 "Add to PATH" 체크), 또는 WSL(Ubuntu) 권장.

## 2. 프로젝트 폴더 & 가상환경
가상환경(venv)은 프로젝트별로 패키지를 격리한다.
```bash
mkdir ddalgiro-trend && cd ddalgiro-trend
python3 -m venv .venv           # 가상환경 생성
source .venv/bin/activate       # 활성화 (Windows: .venv\Scripts\activate)
python -m pip install --upgrade pip
```
> 프롬프트 앞에 `(.venv)`가 보이면 활성화된 것. 새 터미널을 열 때마다 `source .venv/bin/activate` 다시 실행.

## 3. 기본 패키지 설치
```bash
pip install requests python-dotenv
pip freeze > requirements.txt   # 설치 목록 고정(재현성)
```
> 각 가이드에서 필요한 패키지를 추가로 설치하면 `pip freeze > requirements.txt`로 갱신.

## 4. 프로젝트 구조 (권장)
```
ddalgiro-trend/
├── .venv/                  # 가상환경 (git 제외)
├── .env                    # 비밀키 (git 제외!)
├── .env.example            # 키 이름만 적은 템플릿 (git 포함)
├── .gitignore
├── requirements.txt
├── config.py               # .env 로드, 공통 설정
└── src/
    ├── collectors/         # 02~05 수집기
    ├── storage/            # 06 DB
    ├── analysis/           # 07 LLM, 09 스코어링
    └── jobs/               # 08 스케줄 작업
```

## 5. 비밀키 관리 — `.env` (가장 중요)
**API 키를 코드에 직접 쓰지 않는다.** `.env` 파일에 두고 코드는 환경변수로 읽는다.

`.env` (실제 키 — git에 올리지 않음):
```
NAVER_CLIENT_ID=여기에_발급받은_값
NAVER_CLIENT_SECRET=여기에_발급받은_값
YOUTUBE_API_KEY=여기에_발급받은_값
ANTHROPIC_API_KEY=여기에_발급받은_값
DATABASE_URL=postgresql://trend:trend@localhost:5432/trend
```

`.env.example` (템플릿 — git에 올림, 값은 비움):
```
NAVER_CLIENT_ID=
NAVER_CLIENT_SECRET=
YOUTUBE_API_KEY=
ANTHROPIC_API_KEY=
DATABASE_URL=
```

`config.py`:
```python
import os
from dotenv import load_dotenv

load_dotenv()  # .env 읽어 환경변수로 로드

NAVER_CLIENT_ID = os.environ["NAVER_CLIENT_ID"]
NAVER_CLIENT_SECRET = os.environ["NAVER_CLIENT_SECRET"]
YOUTUBE_API_KEY = os.getenv("YOUTUBE_API_KEY")          # 선택 항목은 getenv
ANTHROPIC_API_KEY = os.getenv("ANTHROPIC_API_KEY")
DATABASE_URL = os.getenv("DATABASE_URL", "postgresql://trend:trend@localhost:5432/trend")
```

## 6. `.gitignore` (비밀·잡파일 제외)
```
.venv/
.env
__pycache__/
*.pyc
.DS_Store
data/raw/      # 원문 캐시는 커밋하지 않음(용량·저작권)
```

## 7. git 초기화 & 첫 커밋
```bash
git init
git add .gitignore .env.example requirements.txt config.py
git commit -m "init: 개발 환경 세팅"
```
> ⚠️ `git add .` 전에 반드시 `.gitignore`에 `.env`가 들어있는지 확인. 실수로 키를 커밋했다면 즉시 키를 **재발급**하고 히스토리에서 제거.

## 8. 동작 확인
```bash
python -c "import config; print('OK, NAVER_CLIENT_ID 길이:', len(config.NAVER_CLIENT_ID))"
```
키가 비어 있으면 `KeyError`가 난다 → `.env`에 값이 채워졌는지 확인.
