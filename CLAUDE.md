# CLAUDE.md — 딸기로컴퍼니 TIPS 과제 프로젝트

> 이 파일은 매 세션 시작 시 Claude가 읽는 컨텍스트다. **세션이 중간에 끊겨도 이 파일 + `PROGRESS.md`만 읽으면 작업을 이어갈 수 있도록** 항상 최신 상태로 유지한다.

## 0. 세션 연속성 규칙 (가장 중요)
- **새 세션을 시작하면**: ① 이 `CLAUDE.md` → ② `PROGRESS.md`(진행 현황) → ③ 직전에 작업하던 `docs/` 파일 순서로 읽고 이어간다.
- **작업을 끝낼 때마다**: `PROGRESS.md`의 "현재 상태 / 다음 할 일 / 마지막 결정"을 갱신한다. (이게 핵심 인수인계 지점)
- **중요한 결정이 나오면**: 메모리(`memory/`)와 `PROGRESS.md`에 모두 기록한다.
- 사용자는 한국어로 소통한다. 산출물도 한국어로 작성한다.

## 1. 프로젝트 정체성
- **이 레포의 목적**: 코드를 만드는 게 아니라, **딸기로컴퍼니가 TIPS 과제로 채택되기 위한 안건들의 "구현 방법"을 정리하는 기획·문서 프로젝트**다.
- **딸기로컴퍼니**: 프리미엄 디저트 전문 배송 서비스 (https://ddalgiro.com/). 입점 업체들에게 디저트를 배송·중개하는 플랫폼 성격.
  - 참고 기사: https://www.kdpress.co.kr/news/articleView.html?idxno=138370
- **TIPS**: 민간 투자 주도형 기술창업 지원 프로그램(정부 R&D). 따라서 산출물은 **기술성·혁신성·R&D 요소 + 사업화 논리**를 함께 갖춰야 한다.

## 2. 안건 목록
- **안건 1 (현재 진행 중)**: 국내 디저트 트렌드 동향을 최신 뉴스·블로그 등 인터넷 정보로 정기 취합·분석하여, **딸기로 자체 웹사이트를 통해 입점 업체에게 "곧 성공할 아이템"을 실시간 소개·추천**하는 AI 시스템.
  - → `docs/01_트렌드분석추천시스템_기획.md`
- **안건 2**: 고객(B2C 소비자·B2B 기업 모두)이 **자연어로 디저트를 문의**하면, 딸기로 입점사뿐 아니라 **온라인 수집한 전국의 현재 운영 중인 디저트 업체까지 통합 분석·랭킹**해 추천하는 AI 검색·추천 엔진.
  - → `docs/02_자연어추천랭킹시스템_기획.md`
  - 확정 방향: 비입점 업체까지 추천(신뢰우선→입점전환 깔때기 + 중개·광고 수익화 + 데이터 해자 + 입점사 투명부스트). 랭킹 1차 기준=질의 적합도, 가산=품질·평판/지역·배송/트렌드(안건 1 연계). 안건 1과 인프라 공유.
- **안건은 1·2 두 건으로 확정** (안건 3 없음). 문서 번호 체계 완결: 안건 1=01, 안건 2=02, 공통=03(인터넷데이터)·04(비용가이드).

## 3. 확정된 방향 (사용자 결정사항)
- **문서 톤**: 기술 구현 설계 **+** TIPS 사업계획서 논리를 **통합**한 종합 문서.
- **현실 반영도**: **단계적/현실적**. MVP → 확장 단계로 나눠 초기엔 적은 비용·인력으로 시작 가능하게 설계.
- **개발 인력**: 충분하다고 전제(제약 아님).
- **예산**: 본 분석 결과(04 비용가이드)에 연동해 책정 예정.
- **보유 데이터**: **없음** → 인터넷 공개 데이터만으로 부트스트랩(03 문서). 자사 데이터는 향후 서비스가 생성하는 피드백 데이터로 축적.

## 5. AI/LLM 사용 시 주의
- 이 프로젝트에서 "AI 활용"의 기본값은 **최신 Claude 모델(Opus 4.8 / Sonnet 4.6 / Haiku 4.5)** 의 API를 가정한다.
- 단, 비용 비교용으로 **OpenAI(GPT)·Gemini 단가도 `04 §2`에 병기**한다. 선택 기준은 단가가 아니라 한국어/도메인 품질 PoC.
- LLM 관련 구체 수치(가격·토큰·모델 ID)는 기억에 의존하지 말고 Claude는 `claude-api` 스킬, OpenAI/Gemini는 웹검색/공식 페이지로 확인 후 기재한다.
